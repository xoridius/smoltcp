# Production cleanup verification

Date: 2026-07-21

This report covers the cleanup from
`6a8173cdbadff7dd386218f7876184820dd2f3d5` through the final code revision
`c583bf75105ecdba55a8982be49aaeff59061a56`. Verification tooling is fixed at
`9b33c341049dcba1f2e89b98a0b1dd7d9dc87780`; that commit changes tests,
scripts, and rustdoc, not the measured library or profiler code.

## Result

The cleanup removes 2,322 net lines across 26 files. The profiler examples are
68 lines smaller after moving tests and worker coordination into private
modules; TCP tests and fork-owned state moved out of `tcp.rs` into private
modules.

No production gate regressed. Default and constrained-Apple TCP sockets remain
536 and 592 bytes. Every dynamic traffic cell completed with zero pool charge
after teardown and unchanged net-heap and workload-RSS invariants.

## Cleanup A/B

The control and candidate were clean detached worktrees with byte-identical
`Cargo.lock` files. Runs were pinned to one CPU, used one warm-up, and alternated
five three-second A/B pairs. The landing threshold is -2% throughput. RSS blocks
only when the paired median exceeds both one host page and 1% in at least three
of five pairs.

The two size checks matched every recorded footprint. The full 42-cell
before/after traffic screen matched every pool, net-heap, and workload-RSS
invariant. It is a coverage screen, not a timing verdict. Its five cells below
